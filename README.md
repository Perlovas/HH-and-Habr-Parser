# Анализатор вакансий HeadHunter

Инструмент для сбора, очистки и анализа вакансий HH: зарплаты, навыки, города, динамика, топ компаний. Есть CLI и веб-UI (лоадер, лайтбокс для графиков).

## Возможности
- Поиск вакансий по ключевым словам через api.hh.ru (rate limit 1 rps).
- Сохраняемые поля: id, название, город, работодатель (id/ссылка), дата, ссылка, зарплата (от/до/валюта), ключевые навыки.
- Обработка: удаление дублей, нормализация зарплат (конвертация к RUB фикс. курсами), выделение навыков по списку, фильтр вакансий, где название содержит ключевые слова.
- Аналитика:
  - Распределение зарплат.
  - Топ навыков (частота).
  - Навыки с наивысшей зарплатой (средняя/медиана/95% ДИ, порог 3 вакансии).
  - Топ городов.
  - Динамика публикаций.
  - Топ компаний: средняя/медиана зарплата, число вакансий, доля вакансий с зарплатой, ссылка на компанию.
- Веб-UI: ключевые слова, страницы, город по названию, min-вакансий на компанию; лоадер, лайтбокс графиков.
- Сохранение данных в `data/`, графиков в `output/`.

## Стек
Python 3.9+, requests, pandas, matplotlib, seaborn, Flask, argparse.

## Структура
- `main.py` — CLI.
- `hh_parser.py` — работа с HH API, троттлинг/ретраи, фильтр по названию, User-Agent.
- `data_processor.py` — очистка, нормализация зарплат, извлечение навыков.
- `analyzer.py` — агрегаты по зарплатам/навыкам/городам/компаниям.
- `visualizer.py` — графики (PNG).
- `webapp.py` — Flask UI, лоадер, лайтбокс, статус HH.
- `requirements.txt`, `LICENSE`, `.gitignore`, `data/`, `output/`.

## Установка
```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск (CLI)
```bash
python main.py --keywords "Python разработчик" "Data Scientist" --pages 10 --with-salary
```
Аргументы:  
`--keywords` (обяз.), `--pages`, `--output`, `--skills`, `--details`, `--area`, `--experience`, `--with-salary`, `--limit`.

## Веб-UI
```bash
python webapp.py
```
Открыть `http://localhost:8000`. По умолчанию: 1 страница × 50 вакансий, только вакансии с зарплатой. Есть настройка минимального числа вакансий на компанию.

## Deploy (Render/Railway)
- `Procfile`: `web: python webapp.py`
- Приложение читает порт из `PORT`
- Health-check: `/health`

## Замечания по HH API
- Обязателен заголовок User-Agent.
- Ограничение ~2000 вакансий, 1 запрос/сек.
- Повторы при ошибках с увеличением задержки.

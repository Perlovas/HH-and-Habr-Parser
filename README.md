# Анализатор рынка вакансий HeadHunter

Инструмент для автоматического сбора, очистки, анализа и визуализации вакансий с HeadHunter (API). Показывает распределение зарплат, наиболее востребованные навыки и их влияние на зарплату, топ городов и динамику публикаций. Есть CLI и веб-UI с лоадером и лайтбоксом для графиков.

## Возможности
- Поиск по ключевым словам через api.hh.ru (учёт rate limit 1 rps).
- Сохранение полей: id, название, город, опыт, дата публикации, ссылка, описание, зарплата (от/до/валюта), ключевые навыки/описание.
- Обработка: удаление дублей, нормализация зарплат (конвертация к RUB фикс. курсами), выделение навыков по списку, подсчёт частоты.
- Аналитика:
  - Распределение зарплат.
  - Топ навыков (частота).
  - Топ городов.
  - Динамика публикаций.
  - Самые востребованные навыки с средней/медианной зарплатой (по вакансиям, где навык встречается).
- Веб-UI: ввод ключевых слов, страниц, города по названию (auto-resolve area), опыт; лоадер; кликабельные графики (лайтбокс).
- Сохранение данных в `data/` (JSON/CSV), графиков в `output/` (PNG).

## Стек
Python 3.9+, requests, pandas, matplotlib, seaborn, Flask, argparse, (опц.) bs4/plotly/pymorphy2.

## Структура
- `main.py` — CLI.
- `hh_parser.py` — работа с HH API, троттлинг, ретраи, фильтры (area/experience/only_with_salary), User-Agent.
- `data_processor.py` — очистка, нормализация зарплат, извлечение навыков.
- `analyzer.py` — агрегаты и метрики по зарплатам/навыкам/городам.
- `visualizer.py` — построение графиков (PNG).
- `webapp.py` — Flask UI, лоадер, лайтбокс, статус доступности HH.
- `requirements.txt`, `LICENSE`, `.gitignore`, `data/`, `output/`.

## Установка
```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск (CLI)
```bash
python main.py --keywords "Python разработчик" "Data Scientist" --pages 10 --with-salary
```
Аргументы:
- `--keywords` (обяз.) — список поисковых фраз.
- `--pages` — число страниц HH (по 100 результатов), по умолчанию 1.
- `--output` — путь для сохранения очищенного датасета (`data/vacancies.json`).
- `--skills` — свой список навыков (по умолчанию встроенный).
- `--details` — тянуть полное описание (медленнее, реже используется).
- `--area` — город/регион (название или ID).
- `--experience` — noExperience | between1And3 | between3And6 | moreThan6.
- `--with-salary` — только вакансии с указанной зарплатой.
- При капче HH: пройти по ссылке капчи и повторить запрос.

Графики сохраняются в `output/`: `salary_hist.png`, `top_skills.png`, `top_cities.png`, `timeline.png`.

## Веб-UI
```bash
python webapp.py
```
Открыть `http://localhost:8000`. Ввести ключевые слова, страниц, город названием (не ID), опыт. По умолчанию: 1 страница × 50 вакансий, без детальных запросов (быстрее, меньше капчи), только с зарплатой. Есть лоадер на время запроса и лайтбокс по клику на график. Таблица «Самые востребованные навыки» показывает частоту и среднюю/медианную зарплату по каждой компетенции.

## Замечания по HH API
- Обязателен заголовок User-Agent.
- Ограничение ~2000 вакансий, 1 запрос/сек.
- Повторы при ошибках с увеличением задержки.
